# Security Policy

## Reporting a Vulnerability

Please report suspected security issues by encrypted email to `security@placeholder-domain.example`.

When reporting, include:

- A concise description of the issue
- Affected file paths or configurations
- Reproduction steps, if available
- Potential impact as you understand it
- Your preferred attribution name, if any

Please do not disclose the issue publicly until remediation is complete or the maintainers confirm coordinated disclosure timing.

## Scope

In scope:

- Code in this repository
- Configurations in this repository
- Sanitized templates and reference artifacts maintained in this repository

Not in scope:

- Third-party components such as Nous Hermes
- Third-party components such as OpenClaw
- Third-party components such as Mission Control

Please report vulnerabilities in those upstream projects to their respective maintainers.

## Response Expectations

- Acknowledgement within 48 hours
- Remediation within 30 days, or an explanation if that timeline is not possible
- Coordinated communication if the issue requires a staged fix

## Disclosure and Credit

After remediation, reporters may be acknowledged in a hall of fame or release note, but only with explicit permission.
